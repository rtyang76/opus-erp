# Opus-ERP 整体蓝图

> 本文档定义系统的完整模块清单、业务流程、核心 ER 关系和实现顺序。
> 这是开发的"地图"，所有实现必须以此为准。

---

## 1. 模块清单

### 1.1 系统管理模块 (system)

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 用户管理 | CRUD、密码重置、启用/禁用 | P0 |
| 角色管理 | CRUD、角色-权限关联 | P0 |
| 权限管理 | 菜单权限、按钮权限 | P0 |
| 菜单管理 | 树形菜单、动态路由生成 | P0 |
| 字典管理 | 数据字典维护 | P1 |
| 操作日志 | 全局审计日志 | P1 |
| 登录认证 | JWT Token、刷新机制 | P0 |

### 1.2 基础资料模块 (master)

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 物料档案 | SKU、规格、分类、单位、安全库存 | P0 |
| 物料分类 | 树形分类（原材料/半成品/成品/辅材） | P0 |
| 计量单位 | 主单位、单位换算 | P0 |
| 仓库档案 | 仓库编码、类型、库位 | P0 |
| 供应商档案 | 编码、联系人、付款条款、评级 | P0 |
| 客户档案 | 编码、联系人、收款条款、信用额度 | P0 |
| 会计科目 | 应收/应付/收入/成本科目 | P1 |

### 1.3 采购管理模块 (purchase)

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 请购单 | 部门申请采购需求 | P1 |
| 采购订单 | 主表+明细、审核流 | P0 |
| 到货接收 | 按订单收货、允许超收/欠收 | P0 |
| 采购入库 | 入库单、自动更新库存 | P0 |
| 采购退货 | 退货申请、退货出库 | P1 |
| 应付挂账 | 应付单生成、发票匹配 | P1 |

### 1.4 销售管理模块 (sales)

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 报价单 | 客户报价、有效期 | P1 |
| 销售合同 | 合同号、条款 | P1 |
| 销售订单 | 主表+明细、审核流 | P0 |
| 销售发货 | 发货通知、拣货、出库 | P0 |
| 销售退货 | 退货申请、退货入库 | P1 |
| 应收挂账 | 应收单、对账单 | P1 |

### 1.5 库存管理模块 (inventory)

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 入库管理 | 采购入库、生产入库、其他入库 | P0 |
| 出库管理 | 销售出库、生产领料、其他出库 | P0 |
| 调拨管理 | 仓库间调拨、审批 | P1 |
| 盘点管理 | 盘点计划、录入、盈亏处理 | P1 |
| 报废管理 | 报废申请、审批、出库 | P2 |
| 即时库存 | 多维度库存查询 | P0 |
| 库存台账 | 流水明细、收发存报表 | P0 |

**⚠️ 核心约束：所有库存变动必须通过 InvTransactionService**

### 1.6 生产管理模块 (production)

| 功能 | 说明 | 优先级 |
|------|------|--------|
| BOM管理 | 物料清单、版本、子件用量 | P0 |
| 生产工单 | 工单下达、领料需求计算 | P0 |
| 领料/退料 | 按工单领料、超额控制 | P0 |
| 报工记录 | 工时记录、产出数量 | P1 |
| 完工入库 | 成品入库、工单关闭 | P0 |

### 1.7 财务管理模块 (finance)

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 应收管理 | 应收单、核销 | P1 |
| 应付管理 | 应付单、核销 | P1 |
| 收款单 | 收款记录、分配 | P1 |
| 付款单 | 付款记录、分配 | P1 |
| 对账管理 | 客户/供应商对账单 | P2 |
| 接口预留 | 总账/凭证/税务接口 | P2 |

### 1.8 报表中心 (report)

| 报表 | 说明 | 优先级 |
|------|------|--------|
| 收发存汇总 | 按期间统计物料收发存 | P0 |
| 库存账龄分析 | 库存呆滞分析 | P2 |
| 销售毛利简表 | 按客户/物料统计毛利 | P1 |
| 采购汇总表 | 按供应商统计采购额 | P1 |
| 生产进度表 | 工单进度跟踪 | P1 |
| 应收/应付账龄 | 账龄分析 | P2 |

## 2. 核心业务流程

### 2.1 采购流程

```
┌─────────┐    ┌───────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  请购单  │───▶│ 采购订单  │───▶│ 到货接收 │───▶│ 采购入库 │───▶│ 应付挂账 │
└─────────┘    └───────────┘    └──────────┘    └──────────┘    └──────────┘
                                    │
                                    ▼
                              ┌──────────┐
                              │ 采购退货 │
                              └──────────┘
```

**状态机：**
```
采购订单: DRAFT ──审核──▶ AUDITED ──关闭──▶ CLOSED
                       └──取消──▶ CANCELLED

采购入库: DRAFT ──审核──▶ AUDITED（审核时调用 InvTransactionService）
```

### 2.2 销售流程

```
┌─────────┐    ┌───────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  报价单  │───▶│ 销售订单  │───▶│ 发货通知 │───▶│ 销售出库 │───▶│ 应收挂账 │
└─────────┘    └───────────┘    └──────────┘    └──────────┘    └──────────┘
                                                       │
                                                       ▼
                                                 ┌──────────┐
                                                 │ 销售退货 │
                                                 └──────────┘
```

**状态机：**
```
销售订单: DRAFT ──审核──▶ AUDITED ──发货──▶ SHIPPED ──完成──▶ COMPLETED
                       └──取消──▶ CANCELLED

销售出库: DRAFT ──审核──▶ AUDITED（审核时调用 InvTransactionService）
```

### 2.3 生产流程

```
┌───────────┐    ┌──────────┐    ┌────────┐    ┌──────────┐    ┌──────────┐
│ 生产工单  │───▶│ 领料出库 │───▶│  报工  │───▶│ 完工入库 │───▶│ 工单关闭 │
└───────────┘    └──────────┘    └────────┘    └──────────┘    └──────────┘
                      │
                      ▼
                ┌──────────┐
                │ 退料入库 │
                └──────────┘
```

**状态机：**
```
生产工单: PENDING ──下达──▶ RELEASED ──开工──▶ IN_PROGRESS ──完工──▶ COMPLETED ──关闭──▶ CLOSED
```

### 2.4 库存变动统一通道

```
所有库存变动请求
        │
        ▼
┌─────────────────────────────────────────────────────────┐
│              InvTransactionService                       │
├─────────────────────────────────────────────────────────┤
│ 1. 校验：库存是否充足、单据状态是否合法                    │
│ 2. 计算：根据移动加权平均法计算成本                        │
│ 3. 更新：更新 inv_stock 表的 quantity 和 avg_cost         │
│ 4. 记录：插入 inv_transaction 流水记录（不可变）           │
│ 5. 事务：以上操作在同一数据库事务中                        │
└─────────────────────────────────────────────────────────┘
```

## 5. 模块依赖关系

```
                    ┌─────────┐
                    │ common  │
                    └────┬────┘
                         │
         ┌───────────────┼───────────────┐
         │               │               │
         ▼               ▼               ▼
    ┌─────────┐    ┌─────────┐    ┌─────────┐
    │ system  │    │ master  │    │  report │
    └─────────┘    └────┬────┘    └─────────┘
                         │
         ┌───────────────┼───────────────┐
         │               │               │
         ▼               ▼               ▼
    ┌─────────┐    ┌─────────┐    ┌─────────┐
    │inventory│    │purchase │    │  sales  │
    └────┬────┘    └────┬────┘    └────┬────┘
         │               │               │
         └───────────────┼───────────────┘
                         │
                         ▼
                   ┌───────────┐
                   │production │
                   └───────────┘
                         │
                         ▼
                   ┌───────────┐
                   │  finance  │
                   └───────────┘
```

**调用规则：**
- 上层模块可以调用下层模块的 Service
- 同层模块之间通过 Service 接口调用，不直接调用 Mapper
- `inventory` 模块是核心，被 `purchase`、`sales`、`production` 依赖
